## 1. 트레이딩봇 실패 사례 (Top 5)

### #1 과적합 (Overfitting / Curve-fitting)
- 가장 보편적인 실패 원인. 백테스트에서 뛰어난 성과가 실전에서 무너짐
- ML 기반 펀드 다수가 역사 데이터의 마이크로구조 노이즈와 스테일 가격을 학습해 실전에서 실패
- 교훈: Walk-forward validation 필수, 과최적화 지표(in-sample 대비 out-sample 격차)를 모니터링

### #2 슬리피지 및 수수료 과소평가
- 실제 슬리피지: 유동 자산 0.1~0.6%/주문, 변동성 급등 시 1.5% 초과
- ETH/USDT 그리드봇: 논문상 1%/일 수익 → 수수료·슬리피지 0.8% 차감 후 순이익 거의 없음
- 2024년 $9M dogwifhat 시장가 주문: 주문서 부족으로 60% 슬리피지 발생, $5.7M 손실
- 교훈: 백테스트에 실제 수수료(0.055%) + 어댑티브 슬리피지 모델 반영 (우리 프로젝트 Cycle 164에서 이미 구현)

### #3 레짐 변화 (Regime Change)
- 불장 최적화 전략이 변동성 국면에서 붕괴
- 2024년 비트코인 ETF 뉴스로 15% 급등락 → 기존 봇 신호 마비
- 교훈: 레짐 감지 레이어(trend/ranging/high-vol) 필요, 레짐별 전략 전환 또는 포지션 축소

### #4 API 장애 및 실행 리스크
- 거래소 다운타임, 레이턴시, 부분 체결 → 미관리 포지션 발생
- 2017년 Tyler Capital: 해저 케이블 절단으로 모니터링 단절 (실거래는 지속)
- 2025년 플래시 크래시: 서킷브레이커 없는 봇이 최악 가격에 청산
- 교훈: timeout 처리, 취소 로직, fallback 프로시저, Health check 루프 필수 (NEXT_STEPS #14 항목)

### #5 자본 관리 실패 (Position Sizing / Correlation Risk)
- 오토매틱 트레이딩 계좌의 73%가 6개월 내 실패
- "5종목 분산 DCA" 운영 시 BTC 15% 하락이면 전 종목 10~25% 동반 하락 → 사실상 단일 포지션
- 소자본 전략이 $1M 규모에서 실패: 마켓 임팩트/슬리피지 분포 변화
- 교훈: 자산 간 상관계수 모니터링, 포지션 사이즈를 변동성 기반으로 설정, 50~70%만 봇 운영

## 4. 다시간 앙상블 실증

### 윈도우 크기 최적값

| 윈도우 | 성능 |
|--------|------|
| 7~14일 | 언더피팅 — 트렌드 정보 부족 |
| **30일** | **최적** — 변동성 패턴 포착 vs 노이즈 균형 |
| 60일 | 노이즈 과다, 최근 변화 반응 저하 |
| 90일 | 장기 구조 파악에 유용하나 단기 신호 희석 |

실증 결론: **30일 윈도우가 단일 최적**이지만, 30+60+90일 **앙상블**은 서로 다른 시간 스케일의 정보를 보완해 단일 윈도우 대비 성능 향상 기대 가능.

### Stacking vs Voting vs Blending 비교

| 방식 | 특징 | 성능 |
|------|------|------|
| **Hard Voting** | 다수결, 구현 단순 | 기준선 |
| **Soft Voting** | 확률 가중 평균 | Hard 대비 소폭 우위 |
| **Blending** | holdout set으로 가중치 고정 | Voting보다 유연 |
| **Stacking** | 메타러너가 base 모델 예측을 재학습 | 최고 성능, 구현 복잡 |

Stacking 실증 수치 (고빈도 트레이딩 ScienceDirect 2024 연구):
- 정확도 81.80%, F1 81.49%, AUC-ROC 88.43% — 단일 모델 및 단순 앙상블 대비 유의미하게 우수

XGBoost+LightGBM+CatBoost 스태킹 결합: 단일 모델 대비 5~15% 성능 향상 보고 (Medium, 2024).

### 크립토 앙상블 AI 사례
- 하이브리드 앙상블 + LLM 봇: 1m/1h/1d 캔들 복합 분석, 인트라데이 속도 + 감사 가능성 확보
- 4개 모델 앙상블 (XGBoost, LightGBM, LSTM, Transformer): 10+ 기술지표, 멀티타임프레임 확인, 6개월 백테스트 약 60% TP 적중률

## 5. 우리 프로젝트에 적용할 교훈

### 액션 아이템 (우선순위 순)

**[A1] XGBoost 다시간 앙상블 — Stacking 방식 (NEXT_STEPS #6)**
- 30/60/90일 윈도우 각각 학습 → 메타러너(Logistic Regression or XGBoost)로 스태킹
- 단순 voting 대신 stacking 우선 시도: 실증상 5~15% 추가 성능 향상 기대
- 메타러너 입력: base 모델 3개의 예측 확률 + 레짐 피처

**[A2] Paper→Live 게이트 강화 (이미 설계됨, 실행 필요)**
- 최소 30일(권장 90일) 페이퍼 운영 후 라이브 전환
- 라이브에서 Sharpe, PF, MDD가 페이퍼 대비 ±20% 이내인지 지속 비교 모니터링
- 슬리피지가 백테스트 가정치 초과 시 자동 포지션 축소

**[A3] Health Check + Telegram 알림 구현 (NEXT_STEPS #13, #14)**
- 5분 liveness 루프: API 응답, 포지션 일관성, 데이터 지연 감지
- Critical 이벤트(MDD 초과, API 장애, 포지션 불일치)는 즉시 Telegram 발송
- 이 없으면 라이브 운영 자체가 무의미 — 실패 사례 #4 직접 적용

**[A4] 레짐 필터 추가**
- 레짐 분류(trend/ranging/high-vol) 레이어를 ML 앙상블 앞단에 추가
- 고변동성 레짐에서는 포지션 크기 50% 자동 축소 또는 신호 무효화
- 실패 사례 #3 (레짐 변화)에 직접 대응

**[A5] 상관계수 모니터링**
- 현재 BTC 단일 종목이라 덜 긴급하지만, 다종목 확장 시 필수
- 포트폴리오 레벨 MDD 추적 + 자산 간 상관계수 임계값 초과 시 포지션 조정

## 요약 (200단어 이내)

2024~2026 트레이딩봇 실패 Top 5는 과적합, 슬리피지 과소평가, 레짐 변화, API 장애, 자본 관리 실패다. 자동화 계좌의 73%가 6개월 내 실패하며, Paper→Live 갭의 핵심은 실행 현실(슬리피지, 레이턴시)과 모니터링 부재다.

성공 패턴은 멀티타임프레임 확인, 앙상블 모델, 레짐 적응, 드로다운 제어의 조합이다. Tickeron FLMs는 XGBoost 기반 앙상블로 연 +48%, PF 4.0+를 달성했다.

앙상블 방식 중 Stacking이 최고 성능(정확도 +5~15%)이며, 30일 윈도우가 단일 최적이지만 30/60/90일 앙상블이 상호 보완적이다.

우리 프로젝트 즉시 적용 과제: (1) XGBoost 30/60/90일 Stacking 앙상블, (2) Health Check + Telegram 알림, (3) 레짐 필터 추가. 이 세 가지가 완성되면 Paper→Live 전환 성공 확률이 유의미하게 높아진다.
